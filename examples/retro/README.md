# Example: Solstice — Design Retrospective

This is a complete example of the Designpowers retrospective workflow output for a fictional project: **Solstice**, a personal finance dashboard redesign.

## What's Included

| File | Description |
|------|-------------|
| `design-retrospective.md` | The retrospective document — structured reflection on wins, misses, taste evolution, and carry-forward lessons |
| `design-state.md` | The project's design state at completion — decisions log, handoff chain, debt register, artefact index |
| `taste-profile.md` | The taste profile that guided aesthetic decisions throughout the project |

## How These Files Relate

```
taste-profile.md          design-state.md
     │                         │
     │  (guided decisions)      │  (recorded decisions)
     │                         │
     └────────┬────────────────┘
              │
              ▼
   design-retrospective.md
              │
              │  (feeds learnings back)
              ▼
        design-memory
   (updated for next project)
```

## Key Lessons Demonstrated

1. **Taste profiles prevent aesthetic drift** — the "calm confidence" target held through 8 decisions
2. **Accessibility and taste work together** — the grey→slate shift improved both contrast and warmth
3. **Surprises are data** — the user choosing Lora (serif) over DM Sans revealed that references show quality bar, not exact direction
4. **Fix rounds are diagnostic** — both rounds traced back to accessibility checks running too late in the pipeline
5. **Design debt tracking reveals patterns** — 3 of 7 items affected the low-vision persona, signalling a systemic gap
