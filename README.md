# Designpowers (v2 Beta)

An agent design team you control. 10 agents that run an inclusive design process while you direct. The system learns your taste.

> [!IMPORTANT]
> **Welcome to the v2 Beta!** This branch introduces a massive structural upgrade moving from a pure linear relay model to a **Boutique Design Studio Architecture** featuring dedicated Orchestration and persistent Memory Management.

---

## 🚀 What's New in v2?

### 1. The Hub-and-Spoke Orchestrator (`using-designpowers`)
Instead of agents babbling continuously in a massive chain, a central Orchestrator now routes your tasks to the precise specialist needed (UI, UX, Motion) without polluting the overall context window.

### 2. The Design Lead (Quality Gate)
You no longer have to review every single intermediate file. The **Design Lead** acts as a mandatory gatekeeper, catching token mismatches and accessibility issues autonomously before presenting the polished work to you.

### 3. The Design Ops Memory Manager
All project state (Taste Profile, Design Decisions, Technical Debt) is now safely isolated inside the `memory/` directory and managed strictly by a single agent, keeping your core creative designers entirely stateless and fast.

### 4. Universal Compatibility (`GEMINI.md` + `CLAUDE.md`)
Whether you run your terminal via **Claude Code** or the **Gemini CLI**, the workspace rules and aesthetic tokens automatically sync across platforms.

## ⚙️ Configuration

Customize your brand tokens instantly by editing the `studio.config.yaml` file in the root directory:

```yaml
design_system:
  tokens:
    brand_color: "#004D40"
    typography_primary: "Inter, sans-serif"
```
